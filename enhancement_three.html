Enhancement Three

Enhancement Two: Algorithms and Data Structure

The Contact manager application is a simple application that can be used to store contacts. These include phone numbers, email addresses, Addresses like zip codes and the names of the contact holder. The application can hold as many contacts as possible and can allow a user to view and delete contacts.

The application has a security module. This means that only the user that has an account can view and add contacts. In case user does not have an account then the application will not allow the user to add or view the contact books.

The application uses text files to store its contacts. These files will reside in the local storage locations of the computer, this means that user will need to carry the device that has the application. However I have seen this as a disadvantage, therefore on the next enhancement it will be solved since the application will use a web based database, one would be able to access the contacts as long as he/she has the application.

The use of text files to store data for an application is ok with the current functionality of the application, however there are some disadvantages. Firstly, one can easily see the records In the text, text files cannot be easily secured compared to database softwares, therefore storing records like passwords and user names will make the application data insecure.

Secondly, organizing and managing records in a text files is tiresome. In database softwares we have many third party applications that we can use to easily manage data, organize and easily view data in our database compared to text files.

Further, databases can be centralized and can be shared to by many users. In these scenario, users will have their accounts in the database and they will be able to access and store their contact books. Passwords can be easily secured using MD5 or other hash function.

To enhance this application, I created a web based application, which I tested using XAMPP. This database contains a table to store all users’ accounts. Once a user gets a new application, he creates an account, these details will be created in user table. Also for the contacts books, there is a table which stores these contacts book names, at the end of each book name, the user name is appended, this will allow all user to create book names of their choice without creating duplicates.

Once a user add a contact, the contact details will be added in a table, however the book name will be stored as a field in the table so that the app can use it when selecting requested from a certain book. Example.

Achievements

The above enhancement will make the application secure, reliable and also easy to maintain. It’s easy also to manage and create back up for the application data. Database software systems have methodologies for creating back up for the records and also a way to recover lost data. This is an advantageous compared to text files.

I spent a lot of time doing a research on database systems. There are many software’s in the internet which are free to use. However most of them do not have sufficient documentation. MySQL is most common and it has a vast sources of documentation. One can easily earn and manage the database.

Due to these enhancement, the application is more reliable and more usefully. All the enhancements that I planned about database have been successfully implemented on the application.
